Whack-A-Mole Game

Interactive browser game demonstrating jQuery, event handling, timers, and dynamic DOM manipulation.

Summary

This project involved creating a fully functional Whack-A-Mole game using HTML, CSS, and JavaScript with jQuery. The game features random mole spawning, click detection, scoring system with special moles, smooth animations, and an end-game results screen.

The game uses a 4x4 grid of holes where moles randomly appear. Players click on moles to score points, with regular moles worth 111 points and special rare moles worth 10,000 points. The game ends when 6 or more moles appear simultaneously on screen.

Role & Technologies

  • Role: Individual project
  • Technologies: HTML5, CSS3, JavaScript (ES6), jQuery 3.7.1
  • Duration: 2 weeks

Key Features

  • Dynamic 4x4 grid generation with 16 clickable holes
  • Random mole spawning using intervals and probability
  • Two mole types: regular (111 pts) and special (10,000 pts)
  • Smooth jQuery animations for mole appearance and whacking
  • Real-time score tracking and point display
  • Game-over logic when 6+ moles appear simultaneously
  • Comprehensive end-game statistics screen
  • Image preloading for smooth performance

Technical Highlights

Dynamic DOM Generation

The game board is generated dynamically in JavaScript, creating 16 hole divs with unique IDs. This approach makes it easy to scale the game (change NUM_HOLES constant) and keeps the HTML clean and maintainable.

Event Delegation

Click events are handled efficiently using jQuery's event delegation. Rather than attaching individual event handlers to each hole, a single handler manages all clicks and determines the appropriate action based on the hole's state (occupied vs. empty, regular vs. special mole).

State Management

The game tracks multiple state variables including current moles on screen, points scored, and counts of different mole types whacked. This allows for accurate game logic and comprehensive end-game statistics.

Timer-Based Game Loop

Uses setInterval() to spawn new moles every 500ms, creating continuous gameplay. The timer is properly cleared when the game ends to prevent memory leaks.

CSS Grid Layout

The game board uses CSS Grid for precise positioning of the 4x4 hole layout with proper spacing. This provides a clean, responsive layout that's easy to adjust.

Reflection

Description: What was this assignment about?

This assignment was about integrating JavaScript with the DOM to create an interactive, event-driven application. Unlike previous assignments that focused on forms or simple interactivity, this required managing game state, timers, animations, and user input simultaneously.

The Whack-A-Mole game taught me how to think about time-based events, random spawning logic, and creating smooth user experiences with animations. It was also my first experience building something that felt like a complete, polished game.

Analysis: What design decisions mattered most?

Using jQuery for Animations: Rather than manually manipulating CSS properties, jQuery's fadeIn(), fadeOut(), and slideUp() methods provided smooth, built-in animations. This made the game feel much more polished with minimal code.

Class-Based State Tracking: Using CSS classes like .target and .special to track hole occupancy was elegant. It allowed me to style moles with CSS while also using the classes as flags in my JavaScript logic.

Separation of Concerns: Breaking the game into distinct functions (spawn, click handler, end game, update display) made the code much easier to debug and modify. Each function has a single, clear responsibility.

Game Balance: Setting the loss condition at 6 simultaneous moles creates good tension. Too few would make the game too easy; too many would be overwhelming. The 25% chance for special moles also creates nice excitement without being too common.

Challenges & Solutions

Challenge: Preventing players from clicking the same mole multiple times and scoring repeatedly.

Solution: Used the .target class to track which holes have moles. The click handler checks for this class first and returns early if it's not present, preventing duplicate scoring.

Challenge: Images sometimes appeared to "pop" instead of animating smoothly.

Solution: Implemented image preloading in a dedicated function that runs on page load. This ensures images are cached before the game starts.

Challenge: Moles would sometimes spawn in already-occupied holes.

Solution: Used jQuery's filter() to find only available holes (those without the .target class) before randomly selecting one for spawning.

What I Learned

  • How to use setInterval() and clearTimeout() for game loops
  • Managing multiple pieces of state in a JavaScript application
  • Event delegation patterns for efficient event handling
  • Using CSS classes as state flags in JavaScript
  • jQuery animation methods and chaining
  • Importance of proper cleanup (clearing timers) to prevent memory leaks
  • Random number generation with Math.random() and probability logic

Future Improvements

With more time, I would add:

  • Difficulty Levels: Easy/Medium/Hard modes that adjust spawn rate and max moles
  • High Score System: Use localStorage to persist high scores between sessions
  • Sound Effects: Audio feedback for whacking moles and game over
  • Countdown Timer: Time-based gameplay instead of just mole count
  • Combo System: Bonus points for whacking moles in quick succession
  • Mobile Touch Support: Better touch event handling for mobile devices
  • Animations: More elaborate mole animations (peeking, retreating)
  • Leaderboard: Submit scores to a server for global competition
