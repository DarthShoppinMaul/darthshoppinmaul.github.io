Whack-A-Mole Game

Interactive browser game demonstrating jQuery, event handling, timers, and dynamic DOM manipulation.

Play Game

Summary

This project involved creating a fully functional Whack-A-Mole game using HTML, CSS, and JavaScript with jQuery. The game features random mole spawning, click detection, scoring system with special moles, smooth animations, and an end-game results screen.

The game uses a 4x4 grid of holes where moles randomly appear. Players click on moles to score points, with regular moles worth 111 points and special rare moles worth 10,000 points. The game ends when 6 or more moles appear simultaneously on screen.

Role & Technologies

Individual Project HTML5 CSS3 JavaScript ES6 jQuery 3.7.1 2 Weeks

Key Features

  • Dynamic 4x4 grid generation with 16 clickable holes
  • Random mole spawning using intervals and probability
  • Two mole types: regular (111 pts) and special (10,000 pts)
  • Smooth jQuery animations for mole appearance and whacking
  • Real-time score tracking and point display
  • Game-over logic when 6+ moles appear simultaneously
  • Comprehensive end-game statistics screen
  • Image preloading for smooth performance

Technical Highlights

Dynamic DOM Generation

The game board is generated dynamically in JavaScript, creating 16 hole divs with unique IDs. This approach makes it easy to scale the game and keeps the HTML clean.

Event Delegation

Click events are handled efficiently using jQuery's event delegation. A single handler manages all clicks and determines the appropriate action based on the hole's state.

Timer-Based Game Loop

Uses setInterval() to spawn new moles every 500ms. The timer is properly cleared when the game ends to prevent memory leaks.

How to Play: Click "Start New Game" to begin. Click on moles as they pop up to score points. Regular moles are worth 111 points, special moles (with the mask) are worth 10,000 points! The game ends when 6 or more moles appear at once.
